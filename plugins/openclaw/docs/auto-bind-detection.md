# 自动绑定检测方案（未来规划）

> 状态：**规划中**，尚未实现。仅作为方案记录，供后续迭代参考。

## 背景

当前绑定流程为手动操作：用户在 CoClaw Web 端生成绑定码，然后在 OpenClaw 侧执行 `openclaw coclaw bind <code>`。

未来希望 OpenClaw 实例能自动检测是否已被绑定到 CoClaw，减少手动操作步骤。

## 可能方案

### 方案 A：远程接口轮询

- 插件启动时，若本地无绑定信息，定期向 CoClaw server 查询是否有待绑定请求。
- 优点：不依赖本地文件变化，逻辑集中在 server 侧。
- 缺点：需要 server 提供"待绑定队列"接口；轮询间隔需权衡实时性与资源消耗。

### 方案 B：本地文件监听

- 绑定信息写入 `bindings.json` 后，其他模块通过 `fs.watch` 监听变化。
- 优点：无需网络请求，响应即时。
- 缺点：`fs.watch` 跨平台行为不一致；仅能检测本地绑定变更，无法感知远程状态。

### 方案 C：WebSocket 推送

- 插件与 CoClaw server 保持 WebSocket 长连接，server 在用户发起绑定时主动推送。
- 优点：实时性最好，无需轮询。
- 缺点：要求插件在未绑定状态也保持连接；需设计匿名连接认证机制。

## 决策

暂不实施，当前手动 bind/unbind 流程已满足需求。待用户量增长或产品需求明确后再选型实现。
