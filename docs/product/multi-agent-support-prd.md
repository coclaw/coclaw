# 多 Agent 支持 — 产品设计说明

> 状态：待审核
> 日期：2026-03-14

## 1. 背景

当前 CoClaw 中，每个绑定的 OpenClaw 实例只展示一个默认对话入口（即 main agent）。但 OpenClaw 本身支持用户创建多个独立的 AI 角色（称为 Agent），每个 Agent 可以有不同的名称、头像、技能和工作区。

本次升级让 CoClaw 完整支持这些 Agent，用户可以在 CoClaw 中查看、对话、甚至创建 Agent。

## 2. 核心概念变更

### 2.1 术语调整

| 变更前 | 变更后 | 含义 |
|---|---|---|
| 机器人 | **Claw** | 一个 OpenClaw 实例（用户的本地 AI 服务） |
| （无） | **Agent** | Claw 中的一个 AI 角色，拥有独立的身份和对话 |

一个 Claw 下可以有多个 Agent，就像一台电脑上可以有多个用户账号。

### 2.2 概念关系

```
用户账号
 └── Claw（OpenClaw 实例 A）
 │    ├── 🦞 Main Agent（默认）
 │    ├── 🔬 Research Agent
 │    └── ⚙️ Ops Agent
 │
 └── Claw（OpenClaw 实例 B）
      └── 🤖 Main Agent（默认）
```

## 3. 页面设计

### 3.1 主列表（侧边栏 / 移动端首页）

变更前：每个"机器人"一行，点击进入对话。

```
┌──────────────────────────┐
│  + 添加机器人             │
│  ⚙ 管理机器人            │
├──────────────────────────┤
│  🦞 OpenClaw        ●在线 │  ← 只有一个入口
├──────────────────────────┤
│  会话列表...              │
└──────────────────────────┘
```

变更后：展开为 Agent 列表，按 Claw 分组。

```
┌──────────────────────────┐
│  + 添加 Claw              │
│  ⚙ 管理 Claw             │
├──────────────────────────┤
│  我的 OpenClaw：          │  ← 分组标题（仅多个 Claw 时显示）
│  🦞 Main Agent     ●在线  │  ← 每个 Agent 一行
│  🔬 Research        ●在线  │
│  ⚙️ Ops             ●在线  │
├──────────────────────────┤
│  会话列表...              │
└──────────────────────────┘
```

**交互说明**：

- 点击某个 Agent → 进入该 Agent 的主对话
- Agent 的在线状态继承其所属 Claw 的在线状态
- 只有一个 Claw 时不显示分组标题，视觉上与之前类似
- 只有一个 Agent（main）的 Claw，体验与改版前一致

### 3.2 Agent 头像

Agent 可以拥有自定义头像。头像有三种可能的形态，按优先级展示：

```
┌─────────┐   ┌─────────┐   ┌─────────┐
│  [图片]  │   │         │   │         │
│  .png    │   │   🦞    │   │    M    │
│  .jpg    │   │         │   │         │
└─────────┘   └─────────┘   └─────────┘
  自定义图片      emoji        名称首字母
  （最优先）    （次优先）     （兜底）
```

### 3.3 管理 Claw 页面

变更前：每个"机器人"一张卡片，只显示名称和在线状态。

```
┌─────────────────────────────────┐
│  管理机器人                      │
│                                 │
│  ┌───────────────────────────┐  │
│  │ OpenClaw          ● 在线   │  │
│  │ 更新于 2026-03-14          │  │
│  │                   [解绑]   │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

变更后：卡片内展示 Agent 列表，提供快捷操作。

```
┌─────────────────────────────────────┐
│  管理 Claw                           │
│                                     │
│  ┌─────────────────────────────────┐│
│  │ 我的 OpenClaw            ● 在线  ││
│  │ 更新于 2026-03-14                ││
│  │                                 ││
│  │ Agents:                         ││
│  │ ┌─────────────────────────────┐ ││
│  │ │ 🦞 Main Agent      [对话]   │ ││
│  │ │ 🔬 Research         [对话]   │ ││
│  │ │ ⚙️ Ops              [对话]   │ ││
│  │ └─────────────────────────────┘ ││
│  │                                 ││
│  │ [+ 添加 Agent]         [解绑]   ││
│  └─────────────────────────────────┘│
└─────────────────────────────────────┘
```

**交互说明**：

- "对话"按钮 → 直接跳转到该 Agent 的主对话页面
- Claw 离线时，Agent 列表灰显，"对话"按钮不可点击
- "添加 Agent"按钮 → 弹出创建对话框（见 3.4）

> 界面细节待实施时参照参考项目进一步打磨。

### 3.5 会话列表

会话列表暂保持混合展示（所有 Agent 的会话混在一起），通过 Agent 的 emoji 或名称首字母作为头像来区分来源。

```
┌──────────────────────────────┐
│  会话列表                     │
│                              │
│  🦞 帮我写一个排序算法    ⭐   │  ← Main Agent 的对话
│  🔬 调研 React 状态管理   🧪   │  ← Research Agent 的对话
│  🦞 修复登录页面的 Bug    ⭐   │  ← Main Agent 的对话
│  ⚙️ 检查服务器日志       ⏰   │  ← Ops Agent 的定时任务
└──────────────────────────────┘
```

## 4. 特殊场景处理

### 4.1 只有一个 Agent 的 Claw

绝大多数用户只有 main agent。此时：
- 主列表中该 Claw 下只有一个 Agent 条目，体验与改版前一致
- 管理页面卡片中 Agent 列表只有一行

### 4.2 Claw 离线

- Agent 列表正常展示（使用上次缓存的数据），但标记为离线
- "对话"按钮不可点击
- 点击 Agent 条目时提示"Claw 已离线"

### 4.3 新增 Agent（运行期间）

用户可能在 OpenClaw 本地新增了 Agent，而 CoClaw 页面仍在打开。

- 用户刷新列表时，CoClaw 自动发现新 Agent 并展示
- 新 Agent 的对话会自动创建，无需用户额外操作

## 5. 绑定流程

"添加 Claw"流程与原"添加机器人"完全一致，仅文案变更：

```
步骤 1：点击"添加 Claw"
步骤 2：获取绑定码
步骤 3：在 OpenClaw 中执行绑定命令
步骤 4：绑定成功，自动发现该 Claw 下的所有 Agent
```
