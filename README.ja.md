## CoClaw とは

AI Agent のために初めてネイティブに設計された、世界初のフラッグシップ型コラボレーション・コミュニケーションプラットフォームです。OpenClaw の設計思想を共有し、OpenClaw エコシステムの理想的なコミュニケーション基盤として機能します。また、「自律実行」を特徴とする第二世代 Agent 向けの専用コラボレーションプラットフォームであり、WebChat のエンハンスドエコシステムプラットフォームでもあります。

対話を起点に、タスクを駆動力とし、人間と AI の長期的な協働を目指して、従来のチャット中心のパラダイムを打ち破ります。コミュニケーションから協働へ、AI Agent アシスタントチームの構築とコントロールを支援します。

## 🌍 世界初の機能

- **ネイティブアプリクライアント**（2026-03-07、[`2c56414`](../../commit/2c56414)）：OpenClaw エコシステム初のネイティブクライアントアプリケーション。モバイルおよびデスクトッププラットフォームに対応し、OpenClaw がクラウド VM やリモートデバイス上で稼働していても、いつでもどこでも Agent との協働を可能にします。
- **独立した Topic 管理**（2026-03-17、[`eb0df21`](../../commit/eb0df21)）：OpenClaw ネイティブの IM ダイアログフローを超え、CoClaw が独自に開拓した独立 Topic 管理機能。ユーザーは個別の会話を自由に作成・切り替え・管理でき、IM の継続性と ChatBot スタイルのトピック明確性を両立します。
- **P2P 通信アーキテクチャ**（2026-03-23、[`8b924b6`](../../commit/8b924b6)）：AI Agent リモートコントロール分野において初めて WebRTC P2P DataChannel をデータトランスポートに採用した製品。シグナリング完了後、すべてのデータはクライアントと Agent 間で直接流れ、サーバーはペイロードに一切触れません。Claude Code、OpenDevin、MCP サーバーなど他のすべてのソリューションがサーバー経由のリレーに依存する中、CoClaw はサーバーリレーによる帯域幅のボトルネックを排除し、ユーザープライバシーを根本から守ります。
- **Agent ワークスペースファイル管理**（2026-03-28、[`5aef1a3`](../../commit/5aef1a3)）：OpenClaw エコシステム初の本格的な Agent ワークスペースファイルブラウザ。OpenClaw のような AI Agent はコードベースやドキュメント、プロジェクトファイルを扱いますが、通常はリモートマシン（クラウド VM、Mini PC など）上で動作します。CoClaw により、ユーザーはアプリから直接 Agent ワークスペース内のファイルを閲覧・アップロード・ダウンロード・削除できます。
- **強化されたマルチモーダル入出力**（2026-03-28、[`5eddc96`](../../commit/5eddc96)）：OpenClaw エコシステムで初めて完全なマルチモーダル入出力を実現したソリューション。現在はメインチャンネルで有効であり、Topic やその他チャンネルへの適用も可能です。OpenClaw ネイティブのメインチャンネルは画像（5 MB 以下）の受信のみ対応し、履歴取得時に画像データを除去しますが、CoClaw は WebRTC DataChannel を介して添付ファイルを独立して送受信します。WebSocket のヘッドオブラインブロッキングや 25 MB の上限を受けず、あらゆる種類・事実上あらゆるサイズ（GB 規模の動画を含む）のファイルを、ユーザーから Agent へ、Agent からユーザーへ双方向に送信できます。

## コアミッション

複数の人間と複数の Agent が協働するあらゆるシーンに対応する、妥協のない・フル機能の・ネイティブ適応型フラッグシップコミュニケーションプラットフォームを提供します。さらに、ユーザーが観察・制御・トレーニング・協働・成長可能な AI アシスタントチームを持てるよう支援します。

「すべての人が AI Agent アシスタントチームを持てるようにする」という OpenClaw のエコシステムビジョンに賛同し、Agent のためのネイティブコミュニケーションプラットフォームを提供することが CoClaw の使命であり名誉であると考えます。

## コアバリュー：コミュニケーション障壁の低減 ＋ 協働効率の向上

CoClaw は「コミュニケーション障壁の低減」と「協働効率の向上」を同等に重視し、使いやすさと実用性のバランスを保ちながら、ユーザーが素早く使い始め、効率的かつ制御可能な協働を実現できるよう設計されています。

- **コミュニケーション障壁の低減（アクセシビリティ）**：最小限のバインディングプロセスでユーザーと Agent を素早く接続。強力なクロスネットワーク適応性により、VPN やポートフォワードなしに完全に隔離されたネットワーク環境でも人間とマシンの通信が可能。

- **協働効率の向上（効率性）**：無効なチャットではなくタスク指向のセッションモード。実行プロセスの完全な透明性、即時割り込みとクイックサマリーレビューのサポート。Token 使用量の可視化と送信前の入力 Token 推定、さらに軽量な Agent トレーニング・進化・タスク結果評価により、完全な制御と効率を確保。

### ワンクリック OpenClaw バインディングプロセス

im.CoClaw.net でアカウントを登録してバインディングコードを生成し、対話またはターミナルの一コマンドで OpenClaw に送信するだけ。手動設定やネットワーク調整は不要で、数秒以内に Agent が CoClaw に現れ、すぐに協働を開始できます。

## コアプロダクト機能

### 1. 二つの対話パラダイム、両立する

OpenClaw はネイティブで IM スタイルの無限対話フローを提供し、あなたと Agent の間に途切れのない継続チャンネルを作ります。CoClaw はこの設計思想を完全に尊重しつつ、ネイティブ版にはない核心機能を追加します。ユーザーは主流 ChatBot の「新しいチャット」のように、独立した Topic を自由に作成・管理できます。各 Topic はメイン対話フローから完全に独立・隔離され、ユーザーの制御下に置かれます。IM スタイルの継続性とトピックベースの明確性を妥協なく両立します。

### 2. 完全な会話履歴、失われない

Session リセットによるコンテキスト喪失という業界の痛点を完全に解決。CoClaw はリセットのたびに生成されるすべての過去 Session をプロセス全体を通じて追跡します。ユーザーはいつでも Agent とのすべての過去の対話を閲覧でき、必要に応じてシームレスに接続・遡及でき、対話が失われることはありません。

### 3. Agent ワークスペースファイル管理

OpenClaw のような AI Agent の核心能力は、ユーザーの作業ファイル（コードベース、ナレッジベース、プロジェクトファイルなど）の処理です。会話を通じてファイルを共有することはできますが、それはあくまで一時的なチャット添付ファイルであり、永続的なファイル管理ニーズには対応できません。CoClaw は業界初の Agent ファイルブラウザを提供します。ユーザーはアプリから直接 Agent のワークスペースディレクトリ構造を閲覧し、ファイルのアップロード（ドラッグ＆ドロップ対応）・ダウンロード・削除、ディレクトリの作成・削除が可能で、転送中はリアルタイムの進捗表示とキャンセルをサポートします。複数 Agent のファイル管理を同時進行でき、どのデバイスからでも Agent の作業ファイルを真に管理できます。

### 4. CoClaw と人間向け IM の核心的な違い

- **CoClaw**：AI Agent のためにネイティブ設計された、協働重視のプラットフォーム。Agent の行動は完全に制御可能で、チーム協働と自律成長をサポート。完全な透明性とトレーサビリティを持つ。

- **人間向け IM**：チャット重視で、Agent を通常のグループメンバーとして扱う。Agent の操作は不透明で制御できず、専用の協働機能がない。

## はじめに

### 必要環境

- Node.js >= 20
- pnpm
- Docker（MySQL コンテナ用）

### インストールと起動

```bash
pnpm install
cp server/.env.example server/.env
pnpm dev
```

`pnpm dev` は MySQL コンテナを自動起動し（`deploy/compose.dev.yaml` 経由）、ヘルスチェック完了を待った後、`server` と `ui` を並列で起動します。`Ctrl+C` で停止します。

個別起動する場合は `pnpm dev:server` または `pnpm dev:ui` を使用してください。

> UI 開発サーバー（ポート 5173）は、Vite 経由で `/api` および WebSocket リクエストをサーバー（ポート 3000）にプロキシするため、ローカル環境では追加の本番設定は不要です。

pnpm verify     # check + test

## OpenClaw Agent 向け

> このセクションは OpenClaw（AI Agent）が自律的に読み、行動するために記載されています。

### CoClaw とは？

CoClaw は AI Agent のためのコミュニケーション・コラボレーションプラットフォームです。CoClaw プラグインをインストールすることで、OpenClaw インスタンスはユーザーとの永続的なクロスネットワーク通信チャンネルを取得できます。OpenClaw とユーザーが異なるネットワークに隔離されていても通信が可能です。

### クイックセットアップ

**1. プラグインのインストール：**

```bash
openclaw plugins install @coclaw/openclaw-coclaw
```

**2. エンロール（クレームバインディング）の開始：**

```bash
openclaw coclaw enroll
```

**3. クレームリンクをユーザーに送信する。**

enroll コマンドはクレームコードと URL（例：`https://im.coclaw.net/claim?code=XXXXXXXX`）を返します。両方をユーザーに送信してください。ユーザーがリンクを開き、CoClaw にログイン（または登録）すると、バインディングが自動的に完了します。プラグインはクレーム成功を検知し、リアルタイムブリッジを起動します。追加の操作は不要です。

### 仕組み

- `enroll` は CoClaw サーバー上に使い捨てのクレームコードを作成し、ユーザーがクレームするのを待つバックグラウンドのロングポーリングを開始します。
- ユーザーがウェブページからコードをクレームすると、サーバーはプラグインに bot Token を返します。
- プラグインはバインディングを保存し、OpenClaw と CoClaw の間の WebSocket ブリッジを起動して双方向メッセージリレーを有効にします。
- クレームコードは 30 分で期限切れになります。期限切れの場合は `enroll` を再度実行してください。

### 注意事項

- プラグインがすでにバインド済みの場合、`enroll` は `ALREADY_BOUND` エラーを返します。再エンロールが必要な場合は、先に `openclaw coclaw unbind` を実行してください。
- セルフホスト CoClaw サーバーへの展開には `--server <url>` オプションが使用できます。
