## 布局切换策略

同一套页面内容在移动端与桌面端之间切换入口：

- **桌面端**：左侧 Drawer（主导航）+ 可选 Header。Drawer 展开时可不显示 Header；Drawer 收起/浮层时 Header 出现，左侧按钮控制 Drawer 开关
- **移动端**：底部固定三主导航（对话/机器人/我的）。Drawer 为覆盖层，操作后主动关闭

交互规则：
1. 导航入口统一——不同页面触发同一动作时行为一致
2. 抽屉状态联动——移动端操作后主动关闭
3. 新建对话流程——选机器人 → 进入对话页
4. 顶层页面切换保持轻量，避免嵌套布局

## 设置交互模式

- 无全局"保存"按钮，每项设置变化立即持久化
- 危险操作（清空数据等）二次确认 + loading + 成功反馈

## 错误反馈策略

- 服务端错误优先映射到字段级提示（如 `InvalidLoginId` → 账号框聚焦报错）
- 字段级无法承载时才使用全局 toast

## Bot 列表组织

源项目有系统机器人、创建机器人、分类 tabs。CoClaw 简化为：
- 仅展示用户自己绑定的 bot，无系统机器人
- 顶部主操作：绑定 OpenClaw 实例
- 单列表，无分类 tabs
- 每行：头像、名称、摘要、操作菜单
- 点击主体：进入对话；点击操作按钮：编辑/删除等
